# Memory bank: свадебное приглашение (wedding)

Документ для будущих сессий: контекст репозитория, стек, соглашения и точки внимания при правках.

## Назначение

Статический одностраничный лендинг — приглашение на свадьбу. Язык интерфейса и контента: **русский** (`index.html`: `lang="ru"`).

## Факты о мероприятии (контент)

| Поле | Значение |
|------|----------|
| Дата | 24 июня 2026 |
| Время для таймера обратного отсчёта | 16:00 (зашито в `js/clock.js`) |
| Город | Санкт-Петербург, Россия |
| Адрес | Английская набережная, 28 |
| Имена в футере | Зоя и Матвей |

При изменении даты/времени править **и** текст в HTML, **и** `new Date("June 24, 2026 16:00:00")` в `js/clock.js`.

## Стек и зависимости

- **Разметка**: один файл `index.html`, без сборщика и фреймворков.
- **Стили**: `css/styles.css` (CSS-переменные, sticky-карточки на мобильных, секции программы в виде zigzag).
- **Скрипты** (подключаются с `defer` после GSAP в `<head>`):
  - `js/animations.js` — GSAP + ScrollTrigger: скролл-анимации, параллакс, мобильный snap-scroll, частицы в hero, RSVP-кнопка.
  - `js/clock.js` — обратный отсчёт, flip цифр, конфетти при наступлении даты.
- **CDN** (в HTML): Google Fonts (Great Vibes, Jost), GSAP 3.12.2, ScrollTrigger 3.12.2 с cdnjs.

Сборки npm/yarn в репозитории нет — деплой как статика.

## Карта файлов

```
index.html          — единственная страница, все секции и id
css/styles.css      — глобальные стили, переменные, mobile card stack, zigzag, countdown heart
js/animations.js    — вся логика GSAP / ScrollTrigger / snap на ≤768px
js/clock.js         — таймер и поведение после «нуля»
img/assets/         — hero_top.svg, hero_bottom.svg
docs/superpowers/   — черновики планов/спек (mobile snap, programme zigzag)
```

## Идентификаторы секций (важно для JS)

Порядок для мобильного snap в `animations.js`: `#hero` → `#welcome-message` → `#details` → `#dress-code` → `#rsvp` → `#countdown`.

При добавлении/удалении секции обновить массив `SECTIONS` в `gsap.matchMedia("(max-width: 768px)")` и при необходимости блок sticky в CSS (селекторы `#hero`, `#welcome-message`, …).

## Дизайн-токены (`:root` в `css/styles.css`)

Основные цвета: `--dark-pink`, `--black-pink`, `--light-green`, `--dark-green`, `--white`, `--black`, `--bg-white`, `--bg-green`.

Шрифты: `--header-font` (Great Vibes), `--base-font` (Jost).

Новые акценты лучше выводить из этой палитры для визуальной целостности.

## Поведение, о котором не забывать

1. **`prefers-reduced-motion`**: в `animations.js` при `reduce` на documentElement вешается класс `reduce-motion` и GSAP-анимации не запускаются. Стили для ограничения движения — в CSS (проверять при добавлении анимаций).
2. **Мобильный snap-scroll**: только viewport `max-width: 768px`; кастомная логика `snapTo` с порогом и throttle; точки привязки строятся из `offsetTop` секций; для очень высоких секций добавляется вторая точка внизу.
3. **Параллакс**: `background-attachment: fixed` намеренно не используется на мобильных (комментарии в CSS); фоновые секции используют `background-position` + scrub в GSAP.
4. **Zigzag-линия**: анимация `scaleY` на `.zigzag-line` — линия центрируется без `transform` в базовой вёрстке, чтобы GSAP не конфликтовал с собственным `transform`.
5. **RSVP**: кнопка `.rsvp-btn` — магнитный hover, ripple по клику, пульс; отдельного backend/формы нет (заглушка UX).
6. **После окончания обратного отсчёта**: заголовок меняется на «Мы поженились!», сердце и блок `.countdown` скрываются, один раз запускается конфетти (GSAP).

- Новые интерактивные блоки: по возможности подключать к существующим хелперам `fadeUp` / `staggerGroup` и тем же `ScrollTrigger` паттернам.
